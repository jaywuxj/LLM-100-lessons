## 1. 总论

### 1.1 三大任务的定义

| 任务 | 输入 | 输出 | 核心挑战 |
|------|------|------|----------|
| **文生图 (T2I)** | 文本描述 | 图像 | 语义理解→视觉生成的跨模态转换 |
| **图生文 (I2T)** | 图像 | 文本描述/对话/分析 | 视觉感知→语言表达的跨模态理解 |
| **图生图 (I2I)** | 图像 + 条件 | 编辑/变换后的图像 | 保持语义一致性的精确控制 |

### 1.2 技术发展四个阶段

```
阶段一（2014-2020）：GAN时代
  ├── GAN → cGAN → StackGAN → AttnGAN
  └── 对抗训练，模式崩塌问题突出

阶段二（2020-2022）：扩散模型崛起
  ├── DDPM → DDIM → Latent Diffusion (LDM)
  └── Stable Diffusion v1/v2 发布

阶段三（2023-2024）：Transformer融合扩散
  ├── DiT → SD3(MMDiT) → FLUX.1
  ├── DALL-E 3 (Recaptioning)
  └── 多模态理解：LLaVA → Qwen-VL → InternVL

阶段四（2024-2025）：统一多模态生成
  ├── Chameleon（Meta）、Show-o、Transfusion
  └── 任意模态输入→任意模态输出
```

### 1.3 核心技术组件

| 组件 | 文生图 | 图生文 | 图生图 |
|------|--------|--------|--------|
| **文本编码器** | CLIP / T5-XXL | - | CLIP / T5 |
| **图像编码器** | VAE Encoder | CLIP ViT / SigLIP | VAE / CLIP ViT |
| **核心骨干** | U-Net / DiT / MMDiT | LLM (LLaMA/Qwen) | U-Net/DiT + ControlNet |
| **图像解码器** | VAE Decoder | - | VAE Decoder |
| **模态对齐** | Cross-Attention / adaLN | Linear/MLP Projector | Cross-Attn / IP-Adapter |

```

扩散模型系列（2020-至今）
├── DDPM (2020) → DDIM → Guided Diffusion
├── GLIDE (OpenAI, 2021)
├── DALL-E 2 (OpenAI, 2022) — CLIP + Diffusion Prior
├── Imagen (Google, 2022) — T5-XXL + Cascaded Diffusion
├── Stable Diffusion v1 (2022) — U-Net + LDM
├── SDXL (2023) — Dual Text Encoder, Larger U-Net
├── DALL-E 3 (OpenAI, 2023) — Recaptioning 技术
├── SD3 (Stability AI, 2024) — MMDiT + Rectified Flow
└── FLUX.1 (Black Forest Labs, 2024) — 12B DiT
```

### 2.2 核心架构范式

#### 2.2.1 基于U-Net的潜在扩散模型（LDM/Stable Diffusion）

**核心思想**：在VAE的潜在空间（而非像素空间）进行扩散去噪，大幅降低计算量。

```
架构总览：
  Text ──→ [Text Encoder (CLIP)] ──→ Text Embeddings
                                         │
  Image ──→ [VAE Encoder] ──→ z₀         │
                │                        │
  z₀ + noise → zₜ ──→ [U-Net] ←────────┘ (Cross-Attention)
                         │
            [Iterative Denoising] → z₀_pred → [VAE Decoder] → Image
```

**关键组件**：

1. **VAE**：压缩比8x（512×512→64×64×4），使用重建+KL+LPIPS损失独立预训练
2. **U-Net**：下采样→中间块→上采样+跳跃连接，每块含ResBlock+SpatialTransformer
3. **Text Encoder**：SD1.x用CLIP ViT-L/14, SD2.x用OpenCLIP ViT-H/14, SDXL用双编码器
4. **Cross-Attention注入**：`Attention(Q=image_features, K=text_emb, V=text_emb)`

#### 2.2.2 Diffusion Transformer (DiT)

**核心创新**：用ViT替代U-Net作为扩散模型骨干。

```
DiT Block:
  Patch Tokens + Pos Encoding
       ↓
  [adaLN-Zero(t, c)] → Self-Attention → [adaLN-Zero(t, c)] → FFN
       ↓
  Predicted Noise
```

- **Patchify**：`I×I×C` 分割为 `(I/p)²` 个patch token
- **adaLN-Zero**：通过时间步和条件回归 `(γ, β, α)` 参数，效果最优
- **Scaling Law**：Gflops与FID强负相关，DiT-XL/2在ImageNet 256×256达到FID=2.27

#### 2.2.3 MMDiT（Stable Diffusion 3）

**多模态扩散Transformer，图像和文本使用独立权重但共享注意力。**

```
MMDiT Block:
  Image Tokens → [adaLN] → Q_img, K_img, V_img ─┐
  Text Tokens  → [adaLN] → Q_txt, K_txt, V_txt ─┤
                                                  ↓
                              Joint Self-Attention: Q=[Q_img;Q_txt], K=[K_img;K_txt]
                                                  ↓
                              Split → Image FFN(独立权重) + Text FFN(独立权重)
```

**SD3关键技术**：
- **三重文本编码器**：CLIP-L + CLIP-G + T5-XXL（大幅提升文本理解和文字渲染）
- **Rectified Flow**：`z_t = (1-t)z₀ + tε`，预测速度场 `v = ε - z₀`，路径更直
- **QK-Normalization**：防止注意力权重爆炸
- **Logit-Normal时间步采样**：偏向中间区域（t≈0.5），信息量最大

#### 2.2.4 FLUX.1架构（12B参数）

```
FLUX.1 = MM-DiT Blocks(×19, 双流) + Single-DiT Blocks(×38, 单流)

双流(MM-DiT): Image和Text独立处理 + Joint Attention
单流(Single-DiT): Concat(img,txt) → Self-Attention → Parallel FFN
```

- **Channel Packing**：2×2 patch通道堆叠为16通道，减少token数
- **RoPE位置编码**：替代sin-cos，支持灵活分辨率
- **蒸馏版本**：schnell(1-4步) / dev(25-50步)

### 2.5 训练方案与关键技巧

**标准训练流程**：
```
Phase 0: VAE独立训练 + Text Encoder使用预训练权重 + 预缓存latent/embedding
Phase 1: 低分辨率预训练(256²)，大规模宽泛数据，lr=1e-4, bs=2048+
Phase 2: 高分辨率微调(512²/1024²)，高质量子集，lr=1e-5
Phase 3: DPO/RLHF偏好对齐
Phase 4: 蒸馏(可选)——Guidance/Progressive/Adversarial Distillation
```

**核心损失函数**：

- **ε-prediction**：`L = E[||ε - ε_θ(z_t, t, c)||²]`
- **v-prediction**（SD2.x）：`v = √ᾱ_t·ε - √(1-ᾱ_t)·z₀`
- **Rectified Flow**（SD3/FLUX）：`L = E[||(ε-z₀) - v_θ(z_t, t, c)||²]`
- **Min-SNR Weighting**：`w(t) = min(SNR(t), γ)/SNR(t)`，解决梯度不平衡

**关键训练技巧**：

1. **Classifier-Free Guidance (CFG)**：训练时10%概率丢弃条件，推理时 `ε_guided = ε_uncond + w·(ε_cond - ε_uncond)`，w=7-12
2. **Offset Noise**：加入全局偏移噪声 `noise += 0.1·randn(B,C,1,1)`，改善纯黑/纯白生成
3. **Aspect Ratio Bucketing**：多纵横比桶训练，避免强制正方形裁剪
4. **EMA**：decay=0.9999，用EMA权重推理
5. **Logit-Normal时间步**：`t = sigmoid(randn())`，偏向中间步骤
6. **渐进式分辨率**：256→512→1024，低分辨率学语义，高分辨率学细节
7. **混合精度bf16 + QK-Norm**：DiT架构必须
8. **特征预缓存**：训练前预计算VAE latent和text embedding

## 3. 图生文

### 3.1 技术发展脉络

```
传统(2015-2020): CNN+LSTM → Show and Tell → Show, Attend and Tell
CLIP时代(2021-2023): CLIP → BLIP → BLIP-2(Q-Former)
VLM时代(2023-至今):
├── LLaVA (2023.4) → LLaVA-1.5 → LLaVA-NeXT
├── MiniGPT-4, InstructBLIP
├── Qwen-VL (2023.8) → Qwen2-VL (2024)
├── CogVLM (2023.12)
├── InternVL (2023.12) → InternVL2 (2024)
└── GPT-4V/4o (闭源标杆)
```

### 3.2 通用VLM架构

```
Image → [Visual Encoder(CLIP ViT)] → Visual Features
                  ↓
         [Vision-Language Connector]  ← 类型: Linear/MLP/Q-Former/Cross-Attn
                  ↓
         Visual Tokens (H_v)
                  ↓
Text → [Tokenizer] → Text Tokens → [H_v ; Text Tokens] → [LLM] → Response
```

**Connector对比**：

| 类型 | 代表 | 参数量 | 特点 |
|------|------|--------|------|
| Linear Proj | LLaVA | ~4M | 最简单 |
| MLP Proj | LLaVA-1.5 | ~20M | 两层MLP+GELU |
| Q-Former | BLIP-2 | ~188M | 可学习查询压缩 |
| Cross-Attn | Qwen-VL | ~50M | 保持空间信息 |
| Visual Expert | CogVLM | 大 | 每层插入视觉专家 |

### 3.3 代表模型深度解析

#### LLaVA / LLaVA-1.5

| 配置 | LLaVA | LLaVA-1.5 |
|------|-------|-----------|
| Visual Encoder | CLIP ViT-L/14 224px | CLIP ViT-L/14@336px |
| Connector | Linear | 2-Layer MLP |
| LLM | Vicuna-7B/13B | Vicuna-7B/13B |
| Stage1数据 | CC3M过滤595K | 558K |
| Stage2数据 | LLaVA-Instruct-158K | 665K混合数据 |

**两阶段训练**：

```
Stage 1: 特征对齐 (冻结ViT+LLM, 只训Projector)
  数据: 558K图文对, 格式为单轮对话
  参数: epoch=1, lr=2e-3, bs=128, 8×A100约4小时

Stage 2: 视觉指令微调 (冻结ViT, 训Projector+LLM)
  数据: 665K视觉指令(描述+推理+对话)
  参数: epoch=1, lr=2e-5, bs=128, 8×A100约10-20小时
```

**数据构建**：利用GPT-4基于COCO图像+Caption+BBox生成三类对话——Conversation(58K), DetailedDescription(23K), ComplexReasoning(77K)。

#### Qwen-VL

架构：`Qwen-7B + OpenCLIP ViT-bigG + Position-aware Cross-Attention Adapter`

**三阶段训练**：
```
Stage 1: 大规模预训练 — 1.4B图文对, 224², 全部可训练, lr=2e-4(LLM)/2e-5(ViT)
Stage 2: 多任务预训练 — Caption+VQA+OCR+Grounding混合, 448², 冻结ViT
Stage 3: SFT+RLHF — 高质量对话数据 → Qwen-VL-Chat
```

#### InternVL / InternVL2

- 使用超大ViT(InternViT-6B)
- **Dynamic High Resolution**：图像动态分割为子图，每个独立编码+全局缩略图
- **Pixel Shuffle**降采样visual tokens

#### Qwen2-VL

- **Naive Dynamic Resolution**：不做分割，直接patch化原始分辨率
- **2D-RoPE**编码位置
- 通过min/max_pixels控制token数

|--------|------|------|

| VQAv2 | 1.2M QA | 视觉问答 |

### 3.5 训练技巧

**训练损失**：标准自回归loss `L = -Σ log P(x_i | x_{<i}, I)`，只对助手回复计算。

**关键技巧**：

1. **分阶段冻结**：Stage1只训Connector → Stage2训Connector+LLM → 高级Stage2.5解冻ViT(低lr)
2. **分层学习率**：ViT lr = LLM lr / 5~10
3. **高分辨率策略**：AnyRes分割(LLaVA-NeXT) / Dynamic Resolution(Qwen2-VL) / Pixel Shuffle(InternVL2)
4. **数据混合**：caption 30% + VQA 25% + OCR 15% + reasoning 15% + conversation 15%
5. **LoRA高效微调**：r=128, target=全部线性层, 仅训0.5%参数

## 4. 图生图

### 4.1 技术分类

```
├── 风格迁移: IP-Adapter / LoRA / DreamBooth
├── 图像编辑: InstructPix2Pix / SDEdit / Prompt-to-Prompt
├── 条件控制: ControlNet / T2I-Adapter
├── 图像变体: IP-Adapter / img2img(SDEdit)
└── 超分/修复: SD Upscaler / StableSR
```

### 4.2 核心架构

#### ControlNet

**冻结预训练SD权重，添加可训练副本学习条件控制。**

```
Condition Image → [ControlNet Encoder(SD Encoder可训练副本)]
                         ↓ (Zero Convolution, 初始化为0)
Text + Noise → [Locked SD U-Net] ← ControlNet特征注入(每个skip connection)
                         ↓
                  VAE Decode → Image
```

- **Zero Convolution**：零初始化1×1卷积，训练初期不破坏原模型
- 对小数据集(<50K)和大数据集(>1M)都鲁棒
- 训练：lr=1e-5, 50K-100K步, 1-8×A100

| 条件类型 | 数据 | 规模 |
|---------|------|------|
| Canny边缘 | 任意图+Canny | 50K-3M |
| 深度图 | 图+MiDaS | 50K-3M |
| 人体姿态 | COCO+OpenPose | ~200K |
| 语义分割 | ADE20K/COCO | ~50K |

#### InstructPix2Pix

**文本指令编辑，核心是合成训练数据的pipeline。**

```
数据生成:
  Caption → [GPT-3] → (instruction, edited_caption)
  caption → [SD + 同seed] → Image_before
  edited_caption → [SD + Prompt-to-Prompt] → Image_after
  → CLIP方向一致性过滤 → 454K (before, instruction, after) 三元组
```

- U-Net输入8通道（4ch noise + 4ch condition image latent，新增通道零初始化）
- **双CFG**：Image CFG控制原图忠实度 + Text CFG控制指令响应度
- 训练：454K对, bs=256, lr=1e-4, 10K步, 2×A100

#### IP-Adapter

**解耦Cross-Attention实现图像prompt。**

```
Reference Image → [CLIP Image Encoder] → [Projection] → Image Tokens
在SD每个Cross-Attention层:
  Output = Attn(Q, K_text, V_text) + λ·Attn(Q, K_image, V_image)
只训练Image Projection + 新增K,V线性层，冻结SD原始权重
```

变体：IP-Adapter-Plus(patch features)、FaceID(人脸专用)、InstantStyle(风格迁移)

### 4.3 个性化生成方案

#### DreamBooth
```
3-5张图片 → 训练SD学习 "a photo of [V] dog"
├── Reconstruction Loss: 重建目标对象
├── Prior Preservation Loss: 防止类别漂移(用原模型生成同类图像正则化)
├── lr=1e-6~5e-6, 400-1200步, 1×A100, 5-15分钟
└── 效果好但需微调全部参数
```

#### LoRA微调
```
在U-Net/DiT注意力层注入低秩矩阵: W' = W + α·B·A
├── rank: 4(风格)~128(复杂概念), lr=1e-4~5e-4
├── 10-200张图, 10-50 epochs
├── 1×RTX 4090, 30min-2h, 输出10-200MB
└── 可叠加组合，灵活性最高
```

## 5. 统一多模态生成模型

2024年新范式：将理解与生成统一到单一模型。

| 模型 | 团队 | 架构 | 文本生成 | 图像生成 |
|------|------|------|---------|---------|
| **Chameleon** | Meta | 纯自回归 | Next Token | VQ-VAE离散token |
| **Show-o** | ShowLab+字节 | 混合 | 自回归 | 离散扩散(mask→denoise) |
| **Transfusion** | Meta | 混合 | 自回归loss | 连续扩散loss |

**Show-o创新**：Omni-Attention——文本用Causal Attention，图像用Bidirectional Attention，统一MAGVIT-v2(图像)+LLaMA(文本) tokenizer。

**Transfusion创新**：同一Transformer中，文本token用自回归loss，图像patch用扩散loss，混合在同一batch训练。

```
文生图: SD1.5约2B对 | SDXL约600M | SD3/DALL-E3/FLUX.1估计数十亿级
```

- L3 合成标注(千万级)：Recaptioning/ShareGPT4V
- L4 人工标注(数十万)：COCO/VG，用于SFT/RLHF

| DreamBooth | 1×A100 | 5-15min | ~$1 |

### 训练稳定性清单

1. 梯度裁剪 max_grad_norm=1.0
2. lr预热 warmup=1000-10000步
3. QK-Normalization（DiT必须）
4. bf16 > fp16（更稳定）
5. EMA decay=0.9999
6. Gradient Checkpointing
7. DeepSpeed ZeRO Stage 2/3
8. Flash Attention / xFormers
9. 特征预缓存（VAE latent + Text embedding）

## 8. 未来趋势与展望

1. **统一架构**：理解+生成融入单一模型，Any-to-Any交互
2. **更高效推理**：蒸馏+Consistency Model → 1-4步生成，端侧部署
3. **数据工程升级**：Recaptioning标配化，合成数据质量控制，自动数据配比
4. **可控性增强**：精确空间控制(ControlNet++)、3D一致性、多条件组合
5. **视频生成**：Sora/Veo/Kling，从图像扩展到视频，时序一致性挑战
6. **3D/4D生成**：Text/Image-to-3D，NeRF/3DGS + Diffusion

**关键挑战**：
- 计算成本（12B+模型训练$5-10M）
- 数据版权（LAION面临法律挑战）
- 安全对齐与有害内容防护
- VLM视觉幻觉问题
- 统一评估标准缺失
