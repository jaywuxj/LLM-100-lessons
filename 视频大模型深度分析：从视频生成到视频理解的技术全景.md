## 1. 引言与背景

视频大模型（Video Foundation Models）是当前人工智能领域最具挑战性和商业价值的研究方向之一。不同于图像生成只需处理单帧的空间信息，视频涉及**时间维度上的连续性建模**，对模型的理解能力和生成能力提出了数量级更高的要求。

### 1.1 为什么视频大模型如此重要？

- **数据规模**：一段 1080p、30fps、60 秒的视频包含约 1800 帧图像，数据量约为单张图像的千倍量级
- **时空一致性**：需要同时建模空间语义和时间动态，保证帧间连贯性
- **世界模型潜力**：OpenAI 将 Sora 定位为"世界模拟器"（World Simulator），视频大模型有望成为理解物理世界规律的通用智能基础设施
- **商业价值**：覆盖影视制作、广告创意、电商营销、游戏开发、教育培训等万亿级市场

### 1.2 发展脉络

```
2022          2023              2024                  2025-2026
 │              │                 │                      │
 ├─ Make-A-Video  ├─ Runway Gen-2    ├─ Sora (OpenAI)        ├─ HunyuanVideo 1.5
 ├─ Imagen Video  ├─ Pika 1.0       ├─ Kling (快手)          ├─ Vidu Q1
 ├─ Phenaki       ├─ SVD (Stability) ├─ Runway Gen-3         ├─ Kling 2.0
 │              ├─ VideoLDM       ├─ Veo (Google)          ├─ Sora Turbo
 │              │                 ├─ HunyuanVideo 1.0      ├─ World Model 探索
 │              │                 ├─ CogVideoX             │
```

从最初的短视频片段生成到如今支持分钟级、1080p 甚至 4K 分辨率的高质量视频创作，视频大模型在短短两年内经历了爆发式的技术迭代。

## 2. 视频大模型的核心技术架构

### 2.1 从 U-Net 到 DiT：骨干网络的范式转移

#### 2.1.1 传统 U-Net 架构

早期的视频扩散模型（如 Video Diffusion Models、Make-A-Video）沿用了图像扩散模型中经典的 **U-Net 架构**：

```
输入噪声 z_t ──→ [下采样编码器] ──→ [瓶颈层] ──→ [上采样解码器] ──→ 预测噪声 ε
                    ↕ Skip Connections ↕
```

**U-Net 的局限**：
- 感受野受限于卷积核大小，难以捕获**全局时空依赖**
- 随分辨率增大，计算量增长迅速
- 可扩展性（Scalability）较差，难以通过简单增加参数量提升性能

#### 2.1.2 DiT（Diffusion Transformer）架构

2023 年 William Peebles 和 Saining Xie 提出的 **DiT（Diffusion Transformer）** 彻底改变了这一格局。DiT 用 Transformer 替代 U-Net 作为扩散模型的骨干网络：

```
核心流程：
输入潜变量 z ──→ [Patchify] ──→ [Positional Encoding] ──→ [N × DiT Block] ──→ [Unpatchify] ──→ 输出

DiT Block 内部：
┌──────────────────────────────────────────────────────┐
│  Layer Norm → Multi-Head Self-Attention → Layer Norm │
│            → Feed-Forward Network                    │
│            + AdaLN-Zero 条件注入                      │
└──────────────────────────────────────────────────────┘
```

**DiT 的核心创新**：

| 特性 | 描述 |
|------|------|
| **Patchify** | 将潜变量划分为 patch token 序列，类似 ViT |
| **AdaLN-Zero** | 自适应层归一化，将时间步和条件信息注入每一层 |
| **全局注意力** | Transformer 的自注意力天然支持全局依赖建模 |
| **可扩展性** | 遵循 Scaling Law，增大模型参数可持续提升质量 |

**DiT 变体对比**：

| 模型 | 条件注入方式 | 参数量 | FID (ImageNet 256×256) |
|------|-------------|--------|----------------------|
| DiT-S/2 | AdaLN-Zero | 33M | 68.4 |
| DiT-B/2 | AdaLN-Zero | 130M | 43.5 |
| DiT-L/2 | AdaLN-Zero | 458M | 23.3 |
| DiT-XL/2 | AdaLN-Zero | 675M | **9.62** |

这一结果证明了 Transformer 在扩散模型中的**Scaling Law 特性**——模型越大，生成质量越好。

### 2.2 3D VAE：时空压缩的关键

视频数据量巨大，直接在像素空间进行扩散运算不可行。**3D VAE（3D Variational Autoencoder）** 是将视频从像素空间压缩到低维潜空间的核心组件。

#### 2.2.1 架构设计

```
原始视频 (T × H × W × 3)
    ↓ 3D Encoder (空间+时间下采样)
潜变量 (T/t × H/s × W/s × C)
    ↓ Diffusion Process (在潜空间进行去噪)
去噪后的潜变量
    ↓ 3D Decoder (空间+时间上采样)
生成视频 (T × H × W × 3)
```

**关键参数**：
- **空间压缩比 (s)**：通常为 8×8，将 1024×1024 压缩为 128×128
- **时间压缩比 (t)**：通常为 4×，将 120 帧压缩为 30 个时间 token
- **通道数 (C)**：潜空间通道数，通常为 4-16

#### 2.2.2 主流 3D VAE 方案对比

| 模型 | 空间压缩 | 时间压缩 | 通道数 | 特点 |
|------|---------|---------|--------|------|
| Sora (推测) | 8×8 | 4× | 16 | Spacetime Patches |
| 可灵 | 8×8 | 4× | 16 | 自研 3D VAE，支持 1080p |
| HunyuanVideo | 8×8 | 4× | 16 | 因果 3D VAE (CausalConv3D) |
| CogVideoX | 4×4 | 4× | 16 | 3D因果卷积 |
| SVD | 8×8 | 1× | 4 | 空间压缩为主，无时间压缩 |

#### 2.2.3 因果 3D 卷积（Causal 3D Convolution）

HunyuanVideo 等模型采用的**因果 3D 卷积**是一个重要创新：

```python
# 因果3D卷积的核心思想：
# 对于时间维度，只看当前和过去的帧，不看未来的帧
# 这保证了自回归生成的因果性

class CausalConv3D:
    """
    时间维度：因果padding（只在过去方向padding）
    空间维度：对称padding
    
    优势：
    1. 支持流式生成（streaming generation）
    2. 保持时间因果性
    3. 第一帧可独立编码（兼容图像VAE）
    """
    def forward(self, x):  # x: (B, C, T, H, W)
        # 时间维度因果padding
        x = F.pad(x, (0,0, 0,0, kernel_t-1, 0))  
        return self.conv3d(x)
```

### 2.3 时空注意力机制

视频 Transformer 的核心难题是如何高效地建模时空依赖关系。全注意力（Full Attention）的计算复杂度为 $O((T \times H \times W)^2)$，对于高分辨率长视频来说完全不可行。

#### 2.3.1 三种主流方案

**方案一：分解式注意力（Decomposed Attention）**

```
时空分解：
  时间注意力：每个空间位置沿时间轴做 Self-Attention  → O(T²)
  空间注意力：每帧内部做 Self-Attention            → O((H×W)²)
  交替堆叠：[Spatial Attn] → [Temporal Attn] → [Spatial Attn] → ...
  
代表：Video Diffusion Models, AnimateDiff
优势：计算效率高
劣势：时空交互不够充分
```

**方案二：联合时空注意力（Joint Spatiotemporal Attention）**

```
所有 token 一起做 Self-Attention：
  Attention(Q, K, V) where Q, K, V ∈ R^{(T×H×W) × d}
  
代表：可灵 (Kling)，采用 3D 时空联合注意力
优势：建模能力最强，能捕获复杂运动
劣势：计算量巨大，O((T×H×W)²)
```

**方案三：稀疏时空注意力（Sparse Spatiotemporal Attention, SSTA）**

```
HunyuanVideo 1.5 的创新方案：
  ┌─ 全局空间注意力（每帧内全注意力）
  ├─ 局部时间注意力（滑动窗口）
  └─ 关键帧全局时间注意力（稀疏采样）
  
优势：计算量大幅降低（可在14GB显存部署）
效果：几乎不损失生成质量
```

#### 2.3.2 PAB（Pyramid Attention Broadcast）优化

新加坡国立大学提出的 **PAB** 是一种免训练的加速方案：

```
核心思想：
  观察到视频扩散模型中，相邻去噪步骤的注意力输出高度相似
  → 可以跨步骤广播（复用）注意力结果

三层金字塔策略：
  L1: 空间注意力  → 每步计算（变化大）
  L2: 时间注意力  → 每 2-3 步计算一次（变化中等）
  L3: 交叉注意力  → 每 5-10 步计算一次（变化小）

加速效果：实现接近实时的视频生成，速度提升 3-10×
```

### 2.4 条件控制机制

视频生成需要多种条件输入来引导生成过程：

#### 2.4.1 文本条件（Text-to-Video）

```
文本 "一只猫在沙发上伸懒腰" 
  ↓ Text Encoder (T5-XXL / CLIP)
文本嵌入 [N_tokens × D_text]
  ↓ Cross-Attention / AdaLN
注入到 DiT Block 中引导去噪方向
```

主流文本编码器选择：

| 模型 | 文本编码器 | 特点 |
|------|-----------|------|
| Sora | CLIP + T5(推测) | 多编码器融合 |
| HunyuanVideo | MLLM + CLIP | 用多模态大模型做文本编码 |
| 可灵 | T5-XXL | 大参数文本编码器 |
| FLUX | T5-XXL + CLIP | 双编码器 |

#### 2.4.2 图像条件（Image-to-Video）

```
输入图像 → VAE Encoder → 潜变量
  ↓ 拼接到噪声潜变量的第一帧位置
  ↓ 或通过 Cross-Attention 注入
DiT 基于首帧生成后续帧
```

#### 2.4.3 运动控制

| 控制类型 | 方式 | 代表模型 |
|---------|------|---------|
| 运镜控制 | 相机参数注入 | 可灵、Runway |
| 姿态驱动 | OpenPose / DWPose 骨骼 | HunyuanVideo-I2V |
| 首尾帧约束 | 起止帧潜变量拼接 | 可灵、Pika |
| 轨迹控制 | 运动轨迹嵌入 | DragAnything |
| 风格控制 | LoRA / IP-Adapter | 多数模型支持 |

## 3. 视频生成大模型

### 3.1 OpenAI Sora

**定位**：世界模拟器（World Simulator）

#### 技术架构

```
┌─────────────────────────────────────────────────────────┐
│                    Sora 技术架构                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Text Prompt ──→ [CLIP + T5] ──→ Text Embeddings        │
│                                      ↓                  │
│  Native Resolution     Spacetime    DiT                  │
│  Video Data ──→ [3D VAE] ──→ Latent ──→ (Denoising) ──→ │
│                  Patches    Patches   Blocks             │
│                                      ↓                  │
│                              [3D VAE Decoder]            │
│                                      ↓                  │
│                              Generated Video             │
└─────────────────────────────────────────────────────────┘
```

#### 核心创新：Spacetime Latent Patches

Sora 最重要的创新是 **Spacetime Latent Patches（时空潜变量补丁）** 概念：

1. **原生分辨率训练**：不固定输入分辨率和宽高比，直接在各种原始比例上训练
2. **统一的时空表示**：将视频编码为时空 patch 序列，时间和空间维度统一处理
3. **灵活的生成能力**：可生成不同分辨率、不同宽高比、不同时长的视频

```
传统方案：视频 → 固定裁剪 → 固定分辨率 patch
Sora方案：视频 → 保持原始比例 → 自适应 spacetime patches

优势：
- 训练数据利用率更高（不需要裁剪/缩放）
- 生成灵活性更强
- 构图质量更好（学到了原始比例的构图美学）
```

#### 涌现能力

Sora 展现了多种涌现能力：
- **3D 一致性**：场景中的三维结构在不同视角间保持一致
- **长期一致性**：角色和物体的外观在长视频中保持不变
- **物体恒定性**：物体被遮挡后重新出现时保持原样
- **世界交互**：物体之间的物理交互基本合理（如碰撞、反射）

#### 局限性

- 某些物理规律模拟仍不准确（如液体流动、复杂碰撞）
- 手指、牙齿等细节偶尔出现伪影
- 长视频中可能出现逻辑不连贯

### 3.3 腾讯混元视频（HunyuanVideo）

**定位**：全球最大开源视频生成模型

```
┌──────────────────────────────────────────────────────────────┐
│                HunyuanVideo 技术架构                           │
├──────────────────────────────────────────────────────────────┤
│                                                              │

│                                    ↓                         │

│           1080p / 30fps Video                                │
└──────────────────────────────────────────────────────────────┘
```

#### 版本演进

| 版本 | 参数量 | 关键特性 |
|------|--------|---------|
| HunyuanVideo 1.0 | **130亿** | 全球最大开源文生视频模型 |
| HunyuanVideo 1.5 | **83亿** | SSTA 稀疏注意力，14GB 显存部署 |

#### 核心创新

1. **MLLM 文本编码器**：使用多模态大语言模型作为文本编码器，比 T5 有更深的语义理解能力
2. **双流-单流 DiT 架构**：
   - **双流阶段**：文本 token 和视频 token 分别通过独立的 Transformer 层处理
   - **单流阶段**：将两种 token 拼接，进行联合注意力计算
   - 这种设计平衡了计算效率和跨模态交互
3. **稀疏时空注意力（SSTA）**：HunyuanVideo 1.5 的关键突破，将显存需求从 50GB+ 降至 14GB
4. **开源生态**：提供完整的推理代码、LoRA 训练代码和模型权重

**定位**：开源视频生成基础模型

#### 架构特点


视频大模型不仅包括生成方向，**视频理解**（Video Understanding）同样是重要分支。视频理解大模型旨在让 AI 系统能够"看懂"视频内容，回答关于视频的问题。

### 4.1 VideoLLM 技术范式

#### 基本架构

```
┌──────────────────────────────────────────────────────────┐
│              VideoLLM 通用架构                              │

│                                                          │
│  视频输入 ──→ [视觉编码器] ──→ 视频特征                      │
```

| **音视频融合** | 许多场景需要同时理解画面和声音 | 多模态融合模块 |

## 5. 训练方案与数据工程

### 5.1 数据预处理流水线

视频训练数据的质量直接决定了生成效果。以 HunyuanVideo 和 SVD 为代表，现代视频生成模型采用**多级数据过滤流水线**：

```
┌────────────────────────────────────────────────────────┐
│           视频数据预处理流水线（典型方案）                    │
├────────────────────────────────────────────────────────┤
│                                                        │
│  原始视频数据（~10亿级 clips）                             │
│           ↓                                            │
│  Stage 1: 基础过滤                                      │
│  ├── 分辨率过滤（< 720p 丢弃）                            │
│  ├── 时长过滤（< 2s 或 > 60s 丢弃）                      │
│  ├── 帧率过滤（< 24fps 丢弃）                            │
│  └── 去重（基于 perceptual hash）                        │
│           ↓  (~3亿 clips)                              │
│                                                        │
│  Stage 2: 质量过滤                                      │
│  ├── 美学评分（Aesthetic Score > 阈值）                    │
│  ├── 技术质量（模糊检测、压缩伪影检测）                      │
│  ├── 运动评分（光流分析，过滤静态/过度运动）                  │
│  └── 内容安全过滤（NSFW、暴力等）                          │
│           ↓  (~5000万 clips)                           │
│                                                        │
│  Stage 3: 语义标注                                      │
│  ├── 视频描述生成（Video Captioning，使用VLM）             │
│  ├── 场景分类                                           │
│  ├── 动作标签                                           │
│  └── 质量标签（高/中/低质量标记）                           │
│           ↓                                            │
│  最终训练数据集                                           │
└────────────────────────────────────────────────────────┘
```

**数据标注的关键技术**：

| 标注任务 | 技术方案 | 作用 |
|---------|---------|------|
| 视频描述 | 多模态LLM (如 LLaVA) 生成详细描述 | 提升文本-视频对齐质量 |
| 美学评分 | 基于 LAION-Aesthetic 的视频版美学模型 | 过滤低质量内容 |
| 运动估计 | RAFT / UniMatch 光流模型 | 确保适中的运动幅度 |
| 场景切割 | PySceneDetect / TransNet | 确保每个 clip 是单一场景 |

### 5.2 多阶段渐进式训练

现代视频生成模型普遍采用**多阶段渐进式训练**策略：

```
┌─────────────────────────────────────────────────────────────┐
│                    多阶段训练策略                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  阶段0: 图像预训练                                            │
│  ├── 数据：大规模图像-文本对（数亿级）                           │
│  ├── 目标：学习空间生成能力和语义理解                            │
│  ├── 分辨率：256×256 → 512×512                               │
│  └── 时间：数周                                              │
│                                                             │
│  阶段1: 低分辨率视频预训练                                      │
│  ├── 数据：大规模视频数据（过滤后数千万级）                       │
│  ├── 分辨率：256×256, 16-24帧                                │
│  ├── 目标：学习基本时间动态和运动模式                            │
│  └── 引入时间注意力层（从零初始化或 inflate）                    │
│                                                             │
│  阶段2: 高分辨率视频微调                                        │
│  ├── 数据：高质量精选视频数据（数百万级）                         │
│  ├── 分辨率：512×512 → 1024×1024, 24-64帧                    │
│  ├── 目标：提升画面质量和运动连贯性                              │
│  └── 更低的学习率，更精细的质量筛选                              │
│                                                             │
│  阶段3: 长视频与高清微调                                        │
│  ├── 数据：极高质量视频数据（数十万级）                           │
│  ├── 分辨率：1080p+, 120+帧                                  │
│  ├── 目标：长视频一致性、电影级画质                              │
│  └── 可能使用人类偏好数据进行 RLHF                              │
│                                                             │
│  阶段4（可选）: 特定能力微调                                    │
│  ├── 运镜控制微调                                             │
│  ├── 图生视频微调                                             │
│  └── 人物一致性微调                                            │
└─────────────────────────────────────────────────────────────┘
```

### 5.3 Scaling Law 在视频领域的验证

DiT 论文最重要的贡献之一是验证了 **Scaling Law** 在视频扩散模型中的有效性：

```
Scaling Law 三大维度：

1. 模型规模 (Model Size)
   DiT-S (33M) → DiT-B (130M) → DiT-L (458M) → DiT-XL (675M)
   FID: 68.4   →   43.5    →    23.3     →     9.62
   结论：参数量增大 → 生成质量持续提升

2. 训练数据量 (Data Size)
   数据量增大 → 生成质量提升，但边际收益递减
   关键发现：数据质量比数据数量更重要

3. 计算量 (Compute)
   训练 FLOPs 增大 → 更好的收敛和生成质量
   遵循幂律关系：Loss ∝ C^(-α)

视频领域的特殊发现：
- 时间维度的 Scaling 效果不如空间维度明显
- 数据质量对视频模型的影响远大于图像模型
- 3D VAE 的压缩质量是性能瓶颈之一
```

## 6. 评估体系与基准测试

### 6.1 自动评估指标

| 指标 | 全称 | 评估维度 | 局限性 |
|------|------|---------|--------|
| **FVD** | Fréchet Video Distance | 视频整体质量 | 对时间一致性不够敏感 |
| **FID** | Fréchet Inception Distance | 单帧图像质量 | 不考虑时间维度 |
| **CLIPSIM** | CLIP Similarity | 文本-视频对齐度 | 只评估语义，不评估动态 |
| **IS** | Inception Score | 生成多样性和质量 | 与人类感知相关性低 |

### 6.2 VBench：综合评估框架

VBench 是目前最全面的视频生成评估基准，采用**层次化解耦**设计：

```
VBench 评估维度体系：
├── 视频质量 (Video Quality)
│   ├── 画面质量 (Subject Quality)
│   ├── 背景一致性 (Background Consistency)
│   ├── 时间闪烁 (Temporal Flickering)
│   ├── 运动平滑度 (Motion Smoothness)
│   ├── 动态程度 (Dynamic Degree)
│   └── 美学质量 (Aesthetic Quality)
│
└── 条件一致性 (Video-Condition Consistency)
    ├── 整体一致性 (Overall Consistency)
    ├── 人物一致性 (Human Action)
    ├── 色彩描述 (Color)
    ├── 场景描述 (Scene)
    ├── 物体类别 (Object Class)
    ├── 多物体 (Multiple Objects)
    ├── 空间关系 (Spatial Relationship)
    └── 外观风格 (Appearance Style)
```

自动指标与人类感知存在显著 gap，因此人类评估仍然不可或缺：

```
人类评估维度：
├── 视觉质量 (Visual Quality): 画面清晰度、细节丰富度
```

### 7.2 时间一致性

**问题**：长视频中角色外观、场景背景的一致性仍然是最大挑战之一

```
主要表现：
├── 角色面部/服装在不同帧间微变
├── 背景元素出现/消失
├── 物体数量不一致
├── 光照方向突变
└── 风格漂移（前后帧风格不同）

当前解决思路：
├── 更强的时间注意力机制
├── 显式的 ID 一致性模块（如 IP-Adapter）
├── 首尾帧约束
├── 分块生成 + 重叠混合
└── 后处理校正
```

### 7.3 物理世界模拟

```
已能模拟的物理现象：
✓ 基本的重力效果
✓ 简单的碰撞
✓ 大致的流体运动
✓ 光影反射（大致方向）

仍然困难的场景：
✗ 精确的流体动力学（水花飞溅细节）
✗ 复杂的多体碰撞
✗ 准确的镜面反射和折射
✗ 布料的精细物理模拟
✗ 因果逻辑（A导致B导致C的连锁反应）
```

## 10. 总结

### 10.1 关键技术总结

| 技术方向 | 核心方案 | 成熟度 | 趋势 |
|---------|---------|--------|------|
| 骨干网络 | DiT (Diffusion Transformer) | ★★★★★ | 已成为主流标准 |
| 视频编码 | 3D VAE + 因果卷积 | ★★★★ | 向更高压缩比演进 |
| 注意力机制 | 联合时空注意力 → 稀疏注意力 | ★★★★ | 效率与效果的平衡 |
| 文本编码 | T5/CLIP → MLLM | ★★★★ | 多模态大模型编码器 |
| 训练策略 | 多阶段渐进式训练 | ★★★★★ | 图像预训练→视频微调 |
| 评估体系 | VBench 多维评估 | ★★★ | 向更贴近人类感知演进 |

### 10.2 核心洞察

1. **DiT 是当之无愧的主流架构**：从 Sora 到可灵到混元视频，几乎所有顶尖视频生成模型都采用了 Diffusion Transformer 架构，其可扩展性和全局建模能力是关键优势。

2. **Scaling Law 持续有效，但数据质量更重要**：增大模型参数量和训练计算量可以持续提升生成质量，但视频领域的数据质量（而非数量）是更关键的因素。

3. **从视频生成到世界模型的跨越尚未完成**：当前模型本质上学习的是像素分布的统计规律，而非真正的物理规律。从"看起来像"到"真正理解"物理世界，仍有巨大鸿沟。

4. **开源生态快速成熟**：HunyuanVideo（130 亿参数）、SVD 等重量级模型的开源，极大地降低了视频生成的技术门槛，推动了整个领域的快速发展。

5. **推理效率是商业化关键**：从 HunyuanVideo 1.5 的 14GB 显存部署到 PAB 的免训练加速，推理效率的提升对于商业落地至关重要。

6. **视频理解和视频生成正在走向融合**：未来的统一多模态模型将同时具备视频理解和生成能力，这是通向通用人工智能的重要一步。

### 10.3 一句话总结

> **视频大模型正处于从"技术突破"到"产业落地"的关键拐点。DiT 架构的成熟、Scaling Law 的验证、开源生态的繁荣，共同推动视频 AI 进入了快速产业化阶段。而从视频生成到世界模型的跨越，将是下一个十年人工智能最激动人心的技术前沿。**
